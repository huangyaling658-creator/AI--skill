# 今日计划生成 Skill

## 核心原则

这个 skill 的目标是生成一份**看起来像用户自己写的**计划，不是模板化的 AI 输出。
计划 = 日记碎碎念 + 时间块任务表。缺少任何一部分都不算完整。

---

## 第零步：自动获取上下文（每次必须执行）

在用户说任何话之前，先运行：

```bash
python3 /path/to/daily-plan-skill/scripts/get-context.py
```

脚本返回 JSON，包含：
- `calendar_events`：今日日历事件（时间 + 标题）
- `unfinished_yesterday`：昨日计划中 ○ 未完成的任务
- `yesterday_note_found`：是否找到昨日备忘录

**如何使用这些数据：**

| 数据 | 用途 |
|------|------|
| `calendar_events` | 日历中有事件的时间段**不排其他任务**；重要会议可在碎碎念中提前预告 |
| `yesterday_plan` | 读取昨天计划的完整内容，推断哪些任务可能没完成、哪些方向值得延续，自然融入今天的碎碎念或任务中 |

**如果脚本失败（权限、环境问题）：**
- 跳过自动获取，正常进入第一步
- 告知用户："今天没拿到日历和昨日未完成项，如需接入可运行 `scripts/setup.sh`"

---

## 第一步：获取今日输入

用户会通过语音或文字告诉你今天的大方向。结合**第零步获取的上下文**，从中提取：
- 今天的核心任务/焦点是什么
- 日历事件是否影响时间块安排（已从第零步自动获取）
- 当前的心情/状态（如果提到了）
- 昨天未完成的事（已从第零步自动获取，无需用户重复说）

如果用户没有提供任何输入，则基于第零步上下文 + 用户月目标和风格画像自主推断今日安排。

---

## 第二步：生成计划

读取 `references/style-profile.md` 获取用户风格画像，然后按以下结构生成：

### 计划结构模板

```
[日期，如 3.28]
[2-4句碎碎念：今天的状态、昨天发生了什么、对今天的感受或期待。口语化，真实，不矫情。]

[如果有临时灵感/想法，用【想法】标签插入]

[时间段，如 11-12] [任务]
[时间段] [任务]
...（按时间倒序或正序排列，用户原始计划是倒序居多）

[如果有补充说明，在任务下方加一段口语化的备注]
```

### 用户风格关键规则（必须遵守）

**时间习惯：**
- 工作时段：11:00 - 18:00（主要创作/工作）
- 吃饭：12-13、18-19（固定）
- 运动/健身：19-20 或 20-21
- 洗澡：21 或 21:30
- 深夜副业/娱乐：22:00 - 00:30
- 睡觉：00 或 00:30（不要安排早于22点睡）

**任务风格：**
- 任务描述具体，不模糊。不写"做AI项目"，要写"用suno做3首抖音爆款AI音乐，发布到小红书"
- 括号内加原因/方式，例如：`发小红书 15-16（带黑客松比赛tag，图文格式）`
- 灵感/想法用【想法】标签，如：`【想法】做一个专门分析音乐风格的平台`
- 每天必须有的固定项：吃饭、洗澡、睡觉
- 任务量：8-15个，不要少于8个（用户高密度执行风格）

**完成标记：**
- 生成时任务无前缀符号，直接以时间段开头
- 完成后用户手动标记 ✅ 即可

**碎碎念风格：**
- 口语化，像朋友圈或日记
- 可以包含：对昨天的感想、今天的心情、对某件事的期待或担忧
- 不要说教，不要"今天要加油哦！"这种鸡汤
- 2-4句，不超过100字

**禁止：**
- 不要加"## 工作模块"这种标题
- 不要写"根据您的风格..."这种AI腔
- 不要把任务格式化为表格
- 不要加总结或结尾语

---

## 第三步：处理昨日未完成项

如果 `unfinished_yesterday` 不为空：
- 将未完成项插入计划**最前面的工作时段**，标注 `（昨天没干完）`
- 示例：`11-12 产品工作台（昨天没干完）`
- 若未完成项超过 3 个，先展示列表问用户："昨天这几个没完成，今天都排进去还是选几个？"

---

## 第四步：写入 Apple 备忘录

生成计划文本后，调用 `scripts/write-to-notes.py` 将计划写入备忘录。

```bash
python3 /path/to/daily-plan-skill/scripts/write-to-notes.py "计划标题" "计划正文"
```

脚本会：
1. 在 Apple 备忘录创建一条新笔记，标题为今日日期（如 `3.28 今日计划`）
2. 如果当天笔记已存在，则追加内容而不是覆盖
3. 返回成功/失败状态

**如果脚本执行失败（用户没有配置环境），则：**
- 直接在对话中展示计划
- 告知用户需要运行一次 `scripts/setup.sh` 完成环境配置

---

## 第五步：确认与调整

展示生成的计划后，询问用户：
- "有需要调整的吗？"
- 根据用户反馈修改，修改后重新写入备忘录（覆盖模式）

## 给其他 Agent 的调用说明

此 skill 设计为可被其他 agent 调用。调用方式：

**输入（JSON）：**
```json
{
  "user_input": "今天想搞AI音乐，下午研究suno，晚上剪播客",
  "date": "2026-03-28",
  "output_target": "notes | text | both"
}
```

**输出：**
- 生成的计划文本（Markdown格式）
- 写入备忘录的状态（success/failed）

**GitHub 地址：** `github.com/[用户名]/daily-plan-skill`
